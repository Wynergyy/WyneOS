\# Security Policy



\## Supported Versions



The latest tagged version is considered supported:



\- v10.0-phase10



\## Reporting a Vulnerability



If you discover a security vulnerability in WyneOS, please report it responsibly.



Do not create a public GitHub issue.



At this time, WyneOS does not maintain a public security inbox.

Please submit vulnerabilities privately using GitHub’s

“Report a vulnerability” feature under the repository Security tab.



\## Private Modules



Certain components are intentionally excluded from the public repository:



\- Licence Engine internals

\- Signing keys

\- Telemetry processors

\- Behaviour models

\- Predictive algorithms



\## Disclosure Process



If a vulnerability is confirmed:



1\. A fix will be developed internally.

2\. A patched version will be released under a new tag.

3\. Credit may be given to security researchers where appropriate.
