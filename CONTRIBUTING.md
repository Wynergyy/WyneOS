\# Contribution Guidelines



\## Branching Strategy

\- `main` is always production-ready  

\- Feature branches follow: `feature/description`  

\- Fix branches follow: `fix/description`  



\## Commit Standards

\- Semantic commits  

\- Signed commits required (`git commit -S`)  

\- Clear descriptions  



\## Pull Requests

\- Require review  

\- Must pass linting, build, and security checks  

\- Must include documentation updates when needed
